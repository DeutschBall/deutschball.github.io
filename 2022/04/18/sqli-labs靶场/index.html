extractvalue函数报错注入

在MySQL中的报错查询:

原理:

1.XML_documnet,XPath_string两个参数都需要两个字符串

2.我们传入的第二个参数concat('0',(select group_concat(username) from users)))是一个表达式不是字符串,但是表达式求值之后是一个字符串,

这个表达式嵌套了好几个命令,首先查询users表里的所有用户名,合并成一个字符串返回,然后将刚才的字符串前面拼接一个’0’,然后整个字符串作为参数传递给extractvalue函数

==等参数上的表达式都计算完毕之后,函数才会执行,==但是显然我们后面传入的显然不是一个目录(目录最后需要有斜杠/),因此函数报错,报错的时候回显第二个参数发生错误,并且打印我们给出的第二个参数

而在此之前第二个参数上的表达式已经结算完毕,因此报错回显的是我们希望得到的结果

由于报错信息有32个字的长度限制,因此可以使用substr函数分别回显

Less-17

定义了一个检查输入以防止sql注入的函数,下面研究一下他能干什么

1.get_magic_quotes_gpc()获取magic_quotes_gpc的设置情况

当 magic_quotes_gpc 打开时，所有的 ‘ (单引号), ” (双引号), (反斜线) and 空字符会自动转为含有反斜线的溢出字符。

magic_quotes_gpc设置是否自动为GPC(get,post,cookie)传来的数据中的’”加上反斜线。可以用get_magic_quotes_gpc()检测系统设置。

查阅博客https://blog.csdn.net/xiaoxinshuaiga/article/details/80208427

该设置在php.ini中

该设置只在较早版本的php中出现

在5.2.17nts上有但是5.4.45nts上就没有了

2.mysql_real_escape_string

mysql_real_escape_string() 调用mysql库的函数 mysql_real_escape_string, 在以下字符前添加反斜杠: \x00, \n, \r, \, ', " 和 \x1a.

3.intvalue

intval(mixed $value, int $base = 10): int

通过使用指定的进制 base 转换（默认是十进制），返回变量 value 的 int 数值。 intval() 不能用于 object，否则会产生 E_NOTICE 错误并返回 1。

可见该函数是不容易被绕过的
